# Changelog

All notable changes to `smokeping-config` are documented here.
Format loosely follows [Keep a Changelog](https://keepachangelog.com/);
entries from 0.1.1 onward are generated by [knope](https://knope.tech/)
from conventional commits.

## 0.1.0 (2026-04-16)

Initial release. Rust rewrite of the Node.js CLI, distributed as a
self-contained static binary with the curated catalogue embedded at
compile time.

### Features

- `render <patch>` — apply a patch YAML on top of the bundled or
  user-supplied base catalogue and emit a SmokePing `Targets` file.
- `diff-base <patch>` — report drift between a patch and the resolved
  base catalogue.
- `init` — scaffold a starter `patch.yaml` pinned to the bundled
  catalogue snapshot.
- Base resolution cascade: `--base <file>` → `--base-url <url>` →
  bundled snapshot (embedded via `build.rs` + `include_str!`).
- Drift handling via `--on-drift=<ignore|warn|error>`; exit codes
  `0` (success), `1` (drift error), `2` (invalid flag).
- Static binaries published to GitHub Releases for
  `x86_64-unknown-linux-musl`, `aarch64-unknown-linux-musl`,
  `x86_64-apple-darwin`, `aarch64-apple-darwin`,
  `x86_64-pc-windows-msvc`.
- Source crate published to <https://crates.io/crates/smokeping-config>.
